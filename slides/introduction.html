Introduction

Naked Object - Definition
  • An Object Oriented Architectural Pattern.
  • Focus on the domain design (well suited for DDD).
  • Presentation and Persistence Layers are automatically created from the domain.
  • User access directly to properties and actions through UI (objects appear naked).

Naked Object - Impacts on developers
  • Rapid prototyping.
  • Shorter developement cycles.
  • More fluent domain experts involvement.
  • Easy maintenance...
Naked Object - Impacts on end-users
  • Manipulate high level domain objects (OOUI).
  • Objects can have different representations that are more or less adapted to the situation.
  • More frequent software updates.
Why a new Framework ? - Objectives
  • A general design.
  • Modern web implementation.
  • A Living System.
  • Even more agility.
Why a new Framework ? - Approach
  • Keeping good old concepts.
  • New Type System.
  • Make the domain model being manipulated like any domain entity.
Why a new Framework ? - Approach
